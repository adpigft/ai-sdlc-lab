# Context Flywheel

## Summary

The framework should improve context in a flywheel:

Individual knowledge -> team context -> cross-team context -> organization context

Then repeat:

Observe -> learn -> improve context packages and skills

## Operating Idea

- Individual work produces signals, examples, and lessons.
- Team context packages capture what is reusable.
- Cross-team packages generalize approved patterns and guardrails.
- Organization context raises the shared baseline without losing provenance.

## Improvement Loop

1. Observe agent logs, PR feedback, validation failures, and production behavior.
2. Learn which context packages were useful, stale, incomplete, or unsafe.
3. Improve the packages, rubrics, and skills.
4. Reuse the improved packages in the next delivery cycle.

## Notes

- The flywheel is intended to make context increasingly reusable without turning it into an uncontrolled prompt dump.
- Git remains the source of truth for approved context artifacts.
