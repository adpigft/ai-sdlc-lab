# Demo Scope Guide

## Purpose

Define the demo boundary for the AI-SDLC framework.

This keeps the client demo focused on the validated delivery path and avoids surfacing unfinished enterprise layers as if they were active production capabilities.

## Demo Scope

- Wynxx
- Intent
- Requirements
- Context
- Traceability
- Validation
- Control Tower

## Out Of Scope

- Enterprise Productionization Layers
- Future Architecture
- Future Platform Components

## Notes

- The demo should stay read-only unless a specific write action is explicitly approved.
- Demo scope should emphasize synchronized views, traceability, and governance evidence.
