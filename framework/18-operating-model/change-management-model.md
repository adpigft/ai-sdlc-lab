# Change Management Model

## Purpose

This model defines how framework changes should move from draft to release.

It applies to future framework-level changes, not to current delivery work.

## Change Types

- new skill
- skill update
- context package update
- workflow update
- validation rule update

## Change Lifecycle

Draft -> Review -> Approve -> Release

## Approval Process

- Draft: author prepares the change proposal
- Review: impacted owners and reviewers assess the proposal
- Approve: accountable owner signs off
- Release: approved change is published into the framework

## Notes

- Changes should be small, reviewable, and traceable.
- A framework change should not silently alter existing approved delivery behavior.
