# Token Economics Model

## Purpose

This model defines future cost visibility for AI-assisted delivery.

It is intended for planning and governance only.

## Cost Dimensions

- model cost
- token cost
- MCP cost
- infrastructure cost
- review cost

## KPIs

- token per feature
- token per role
- token per defect
- token per release

## Budget Guardrails

- squad: enforce a budget for a team or squad workload
- project: enforce a budget for a delivery project
- portfolio: enforce a budget for multiple projects or programs

## Notes

- The model is about visibility and control, not billing implementation.
- Token economics should be measured alongside quality and delivery outcomes.
