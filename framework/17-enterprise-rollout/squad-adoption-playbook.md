# Squad Adoption Playbook

## Purpose

This playbook describes how different roles adopt the future enterprise rollout model.

## PO

- Responsibilities: business prioritization, value framing, approval alignment
- Skills required: intent, scope control, risk awareness, review discipline
- Onboarding path: foundations, intent, governance, rollout model
- Common mistakes: approving unclear scope, skipping impact checks

## BA

- Responsibilities: requirements clarity, traceability, acceptance quality
- Skills required: requirements, context review, review discipline
- Onboarding path: intent, requirements, traceability, governance
- Common mistakes: incomplete requirements, weak acceptance criteria

## SA

- Responsibilities: architecture boundaries, integration design, change impact
- Skills required: design, API review, context and dependency analysis
- Onboarding path: design, impact analysis, PR review, release governance
- Common mistakes: underestimating compatibility and dependency risk

## Dev

- Responsibilities: implementation quality, tests, traceability, maintainability
- Skills required: implementation, validation, PR review
- Onboarding path: standards, testing, spec-aware review, release readiness
- Common mistakes: coding before approval, weak test coverage

## QA

- Responsibilities: validation strategy, coverage, defect detection
- Skills required: test design, validation, traceability review
- Onboarding path: test-design, validation, observability, drift awareness
- Common mistakes: validating only happy paths, missing evidence gaps

## DevOps

- Responsibilities: deployment readiness, monitoring, operational controls
- Skills required: release, observability, change risk management
- Onboarding path: release, operating model, compliance, rollback readiness
- Common mistakes: under-documenting rollout and monitoring requirements

## PM

- Responsibilities: delivery flow, intervention, escalation, stakeholder visibility
- Skills required: governance literacy, workflow awareness, prioritization
- Onboarding path: Control Tower, adoption model, rollout KPIs, change control
- Common mistakes: relying on status alone without evidence

## Notes

- The playbook is meant for adoption, not enforcement.
- Teams should move through the path in a way that matches their risk and maturity.
