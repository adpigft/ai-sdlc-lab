# Integration Test Design Skill

## Purpose
Design integration tests for external systems, adapters, databases, events, and failure scenarios.

## When to use
Use when integration coverage is needed before implementation, validation, or release.

## Inputs
- spec
- context
- APIs
- traceability
- architecture decisions

## Process
1. Identify integration points.
2. Define success and failure paths.
3. Define data and event dependencies.
4. Map tests to requirements.
5. Return an integration test plan or test artifacts.

## Outputs
- integration test plan or test artifacts

## Quality checks
- external system coverage
- adapter coverage
- database coverage
- event coverage
- failure scenario coverage

## Human gate
Human approval is required before test execution.
