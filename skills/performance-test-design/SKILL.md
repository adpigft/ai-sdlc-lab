# Performance Test Design Skill

## Purpose
Design performance, load, stress, soak, and capacity tests.

## When to use
Use when performance or capacity coverage is needed before validation or release.

## Inputs
- spec
- context
- NFRs
- traceability

## Process
1. Identify performance goals and constraints.
2. Define load profiles and thresholds.
3. Define stress and soak coverage.
4. Return a performance test plan.

## Outputs
- performance test plan

## Quality checks
- load
- stress
- soak
- capacity
- threshold coverage

## Human gate
Human approval is required before performance testing.
