# Dependency Management Skill

## Purpose
Track cross-team, vendor, system, and environment dependencies.

## When to use
Use when dependency ownership and timing need active management.

## Inputs
- dependency list
- plan
- risks
- owners

## Process
1. Identify dependencies.
2. Assign owners and dates.
3. Track status and blockers.
4. Return a dependency matrix.

## Outputs
- dependency matrix

## Quality checks
- ownership identified
- due dates identified
- blockers visible

## Human gate
Human review is required before dependency commitments are finalized.
