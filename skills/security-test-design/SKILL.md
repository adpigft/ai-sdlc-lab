# Security Test Design Skill

## Purpose
Design security tests for authentication, authorization, input validation, secrets, audit, and abuse cases.

## When to use
Use when security coverage is needed before implementation, validation, or release.

## Inputs
- spec
- context
- APIs
- security standards
- traceability

## Process
1. Identify security controls and attack surfaces.
2. Define abuse and negative scenarios.
3. Map tests to requirements.
4. Return a security test plan.

## Outputs
- security test plan

## Quality checks
- authentication
- authorization
- input validation
- secrets
- audit
- abuse cases

## Human gate
Human approval is required before security validation.
