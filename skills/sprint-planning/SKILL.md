# Sprint Planning Skill

## Purpose
Convert approved slices and work items into sprint backlog with capacity, dependencies, and risks.

## When to use
Use when approved work needs to be scheduled into sprint execution.

## Inputs
- approved slices
- work items
- capacity
- dependencies
- risks

## Process
1. Review approved scope.
2. Check capacity and dependencies.
3. Order work items for delivery.
4. Return a sprint plan.

## Outputs
- sprint plan

## Quality checks
- approved scope only
- capacity fit
- dependency fit
- risk visibility

## Human gate
Human approval is required before sprint commitment.
