# Incident Analysis Skill

## Purpose
Analyze production incidents, classify severity and impact, identify root cause, and propose corrective and preventive actions.

## When to use
Use after a production incident has been detected or resolved.

## Inputs
- incident report
- logs
- traces
- metrics
- support tickets
- related artifacts

## Process
1. Classify severity and impact.
2. Reconstruct the incident timeline.
3. Identify root cause and contributing factors.
4. Propose corrective and preventive actions.
5. Return an incident analysis report.

## Outputs
- incident analysis report

## Quality checks
- severity classification
- impact classification
- root cause evidence
- corrective action
- preventive action

## Human gate
Human review is required before remediation planning.
