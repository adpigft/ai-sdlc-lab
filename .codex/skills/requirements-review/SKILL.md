# Requirements Review Skill

## Purpose
Review requirements for completeness, ambiguity, testability, scope, missing NFRs, and acceptance criteria.

## When to use
Use when intent or specification needs review before architecture, design, or implementation work starts.

## Inputs
- intent
- spec
- traceability

## Process
1. Review scope and boundaries.
2. Review completeness and ambiguity.
3. Review testability and acceptance criteria.
4. Review missing NFRs and dependencies.
5. Return review findings only.

## Outputs
- review findings

## Quality checks
- completeness
- ambiguity
- testability
- scope
- NFR coverage
- acceptance criteria coverage

## Human gate
Human approval is required before downstream design work.
