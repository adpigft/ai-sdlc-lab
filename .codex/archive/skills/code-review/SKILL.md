# Code Review Skill

## Purpose
Review code for architecture compliance, security, correctness, maintainability, and test coverage.

## When to use
Use when code is ready for review before merge, release, or promotion.

## Inputs
- source code
- tests
- traceability
- architecture artifacts

## Process
1. Review architecture alignment.
2. Review correctness and maintainability.
3. Review security and test coverage.
4. Return code review findings only.

## Outputs
- code review findings

## Quality checks
- architecture compliance
- security
- correctness
- maintainability
- test coverage

## Human gate
Human review is required before merge or release.
